On simplicity

Engineers rediscover the same lesson every generation: the cheapest code to maintain is the code that was never written. The following quotation captures the idea more memorably than most textbooks manage to.

Everything should be made as simple as possible, but not simpler.

The trap is the second clause. Oversimplifying hides essential complexity rather than removing it, and the hidden complexity resurfaces later as a bug that is far more expensive to find.
